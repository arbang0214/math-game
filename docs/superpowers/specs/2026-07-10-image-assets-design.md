# 이미지 에셋 개편 설계 (UI 전면 개편 1/4, 2026-07-10)

UI 전면 개편의 첫 서브 프로젝트. 마스코트 토끼를 AI 생성 고품질 이미지(PNG)로
교체하고, 상황 컷 2장과 파스텔 배경을 추가한다. 이후 서브 프로젝트(시작 화면,
인게임 다듬기, 게임오버/리더보드 재구성)는 이 에셋 방향 위에서 별도 스펙으로
진행한다.

## 결정 사항 (브레인스토밍 확정)

- 토끼는 AI 생성 이미지로 교체 — **사용자가 생성**(제가 프롬프트 작성·검수·통합)
- 캐릭터 정체성 유지: 흰 토끼, 볼 통통, 반짝 단추 눈, 오른쪽 귀 밑 분홍 리본
  (2026-07-08 v8 스펙의 정체성 계승 — SVG 좌표는 폐기되지만 인상은 유지)
- 당근 목숨 아이콘은 **기존 SVG 유지** (작은 아이콘은 벡터가 또렷)
- 게임 규칙·로직 변경 없음 (표시 전용)

## 에셋 목록

`assets/` 폴더 신설. 전부 사용자 생성, 파일명은 통합 시 아래로 정규화:

| 파일 | 내용 | 요구사항 |
|---|---|---|
| `assets/rabbit-idle.png` | 기본 표정 | 투명 배경, 정사각형, 원본 1024px 내외 |
| `assets/rabbit-happy.png` | 정답 — 활짝 웃음 | 〃 (마스터와 같은 캐릭터·구도) |
| `assets/rabbit-dizzy.png` | 오답 — 어질어질 | 〃 |
| `assets/rabbit-worried.png` | 초조 — 식은땀 | 〃 |
| `assets/rabbit-cheer.png` | 신기록 축하 — 만세 포즈 | 〃 |
| `assets/rabbit-sad.png` | 일반 게임오버 — 귀 처진 시무룩 | 〃 |
| `assets/background.png` | 파스텔 하늘·구름·별 배경 | 세로형(모바일 기준), 텍스트 가독성 위해 은은한 톤 |

- **투명 배경 필수** (배경 제거 후처리는 범위 밖 — 생성 단계에서 요구)
- 표정 4종은 같은 구도(정면, 같은 크기)여야 전환이 자연스럽다

## 제작 워크플로

1. 마스터 프롬프트 전달 → 사용자가 **마스터 1장** 생성·확정 (이 단계에서만 반복)
2. 마스터를 **레퍼런스 이미지로 첨부**해 표정 3장 + 컷 2장 + 배경 생성
   (레퍼런스 기반 편집을 지원하는 도구 필수 — 캐릭터 일관성 확보)
3. 파일 전달(예: D:\) → 검수(같은 캐릭터인가, 투명 배경인가, 구도 일치인가)
   → `assets/`로 정규화·통합 → 브라우저 확인
4. 검수 탈락 컷은 해당 컷만 재생성 (마스터는 유지)

## 통합 방식

### 마스코트 (표정 4종) — "4장 겹침" 방식

- `#mascot` 내부 SVG를 `<img>` 4장으로 교체:
  ```html
  <div id="mascot" data-face="idle">
    <img class="face face-idle" src="assets/rabbit-idle.png" alt="">
    <img class="face face-happy" src="assets/rabbit-happy.png" alt="">
    <img class="face face-dizzy" src="assets/rabbit-dizzy.png" alt="">
    <img class="face face-worried" src="assets/rabbit-worried.png" alt="">
  </div>
  ```
- 기존 CSS 전환 규칙(`#mascot[data-face="..."] .face-...`)과 game.js의
  `dataset.face` 코드를 **무변경**으로 재사용. 4장이 문서에 존재하므로
  프리로드가 자연히 해결되고 전환 깜빡임이 없다 (겹침은 absolute 배치).
- `alt=""` + 기존처럼 장식 취급 (게임 상태는 텍스트로 이미 전달됨)
- 흔들림·점프 연출은 `#mascot` div 대상이라 그대로 동작

### 게임오버 컷 2종

- 게임오버 패널 상단에 `<img id="gameover-cut" alt="">` 1장 추가.
  game.js가 신기록이면 `rabbit-cheer.png`, 아니면 `rabbit-sad.png`로 src 지정
  (기존 `isNewBest` 분기 재사용 — 규칙 로직 아님, 표시 전용)

### 배경

- `body`의 CSS 그라데이션을 `background-image: url(assets/background.png)`로
  교체하되, 이미지 로드 전·실패 시를 위해 **기존 그라데이션을 뒤에 겹쳐 유지**
  (`background: url(...), linear-gradient(...)`)
