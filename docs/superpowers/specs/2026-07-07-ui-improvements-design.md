# UI 개선 3종 설계 (2026-07-07)

플레이 테스트에서 나온 UI 개선 포인트 3건. 전부 표시 전용 변경으로,
게임 규칙(game-core.js, problems.js)은 건드리지 않는다.

## 배경

- 보기 버튼 묶음이 `#choices { margin-top: auto }`로 화면 바닥에 붙어 있어,
  이지선다(1줄)일 때 하단으로 치우쳐 보인다.
- 시간 30% 이하에서 토끼가 살짝 흔들리는 `urgent` 연출은 있으나 표정이
  평소(idle) 그대로라 초조함이 전달되지 않는다.
- 하트 `❤` 문자가 텍스트 글리프로 렌더링되며 CSS 잉크색(#3d3d3d)을
  상속받아 어둡게 보인다 — 가시성 나쁨의 원인.

## 1. 보기 영역 항상 중앙 배치

"보기(#choices) + 피드백(#feedback) + 다음 버튼(#next)" 묶음을 문제 아래
남는 세로 공간의 **중앙**에 배치한다. 모바일·PC 공통(항상 중앙).

- 방법: 세 요소를 `<div id="answer-area">`로 감싸고 래퍼에
  `margin-block: auto`(flex 자동 마진 분배)를 준다. 래퍼 없이 개별 요소에
  auto 마진을 주는 방식은 `#next`가 `hidden`일 때 마진이 함께 사라져
  단계에 따라 위치가 튀므로 쓰지 않는다.
- 피드백 문구는 보기 바로 아래 유지 — 시선 이동 없음.
- 이지선다/사지선다 어느 쪽이든 중앙이라 문제 유형 전환 때 위치가 크게
  튀지 않는다.

## 2. 토끼 단계적 초조 연출

타이머 색 단계와 토끼 상태를 맞춘다:

| 남은 시간 | 타이머 | 토끼 표정 | 토끼 모션 |
|---|---|---|---|
| 100~60% | 초록 | idle | 없음 |
| 60~30% | 노랑 | **worried(신규)** | 느리고 작은 좌우 흔들림 |
| 30% 이하 | 빨강 | worried | 빠르고 큰 흔들림(기존 urgent 강화) |

- SVG에 네 번째 표정 `face-worried` 추가: 처진 눈썹 + 물결 입 + 땀방울.
  기존 표정 3종과 같은 방식으로 겹쳐 두고 `data-face`로 전환한다.
- 판정은 game.js `render()`가 이미 계산하는 ratio를 재사용 — 클래스와
  `data-face` 토글만 추가한다. 표정 우선순위: question 단계에서만 worried
  적용(feedback/gameover 표정 로직은 그대로).
- `prefers-reduced-motion`: 흔들림 애니메이션은 끄되 **표정 변화는 유지**
  — 모션 없이도 긴박함이 전달된다.

## 3. 하트 가시성 개선

- `❤`/`🤍` 문자열 대신 `<span class="heart">♥</span>` 목록으로 렌더링.
  잃은 목숨은 `.heart.off`. U+FE0E(텍스트 프레젠테이션 강제)로 이모지
  렌더링을 막아 어느 브라우저에서든 CSS 색이 먹게 한다.
- 남은 목숨: 선명한 빨강(`--red`) + 옅은 그림자. 잃은 목숨: 연한 회색.
- 크기 1.4rem → 1.6rem.
- 마크업을 다시 짜는 김에 `#hearts`에 `aria-label`(예: "목숨 2개") 부여.

## 테스트 / 검증

- 순수 로직 변경 없음 → 새 단위 테스트 없음. 기존 테스트 76개가 회귀 안전망.
- `npm test` 통과 확인 후 브라우저에서 3항목 육안 확인:
  1. 이지선다·사지선다 모두 보기 묶음이 세로 중앙에 오는가
  2. 노랑 구간에서 worried 표정+느린 흔들림, 빨강 구간에서 빠른 흔들림인가,
     모션 최소화 설정에서 표정만 남는가
  3. 하트가 빨갛고 또렷한가, 잃은 하트가 회색으로 구분되는가

## 범위 제외

- 유형별 시간/배점 차등, 전체 리더보드 (기존 미결사항 그대로)
- 분수 표기 aria-label(`renderWithFractions`)은 별도 후속 항목으로 유지
  — 이번엔 하트 aria-label만 포함.
