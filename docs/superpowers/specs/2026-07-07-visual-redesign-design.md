# 화면 디자인 개선 설계 (팝 카툰 + 리본 토끼)

- 날짜: 2026-07-07
- 상태: 사용자 승인 완료 (비주얼 컴패니언으로 시안 확정)
- 범위: UI(index.html, 신설 style.css, game.js의 표시 계층)만. 게임 규칙 로직 불변.

## 배경 / 목표

초등 5~6학년 대상 분수/소수 타임어택 게임의 화면을 "타임어택 아케이드 게임"답게
개선한다. 현재 화면은 흰 배경 + 기본 폰트의 미니멀 스타일로 긴장감·게임 느낌이
부족하다.

### 설계 제약: 양을 드러내는 그림 금지

피자 조각·막대처럼 수의 크기를 시각적으로 보여주는 표현은 비교(compare) 문제의
정답을 노출하므로 쓰지 않는다. 캐릭터화는 장식(마스코트·표정·색)으로 한정한다.

## 비주얼 테마

- 배경: 하늘색(#cdeaff 계열) → 연분홍(#fff6fa 계열) 세로 그라데이션
- 색은 CSS 커스텀 프로퍼티로 정의 (`--pink`, `--pink-deep`, `--sky`, `--yellow` 등)
- 버튼/카드: 흰 바탕 + 파스텔 테두리 + 통통한 아래 그림자(`box-shadow: 0 6px 0 …`),
  `:active`에서 그림자가 줄며 가라앉는 프레스 효과
- HUD: 하트 ❤/🤍, 점수는 노란 칩(⭐ 점수 🔥x배율), 타이머 바는 잔량에 따라
  초록→노랑→빨강
- 분수는 `3/5` 텍스트 대신 세로 분수 표기(분자·가로줄·분모)로 렌더링.
  problems.js는 그대로 두고 표시할 때 변환한다: 순수 모듈 `format.js`의
  `fractionSegments(text)`가 문자열 안의 `숫자/숫자` 패턴을 세그먼트로 쪼개고,
  game.js가 보기 버튼·question·피드백 모두 이 한 가지 경로로 그린다
  (표시 전용 변환이며 정답 판정과 무관)

## 리본 토끼 마스코트

- 확정 시안의 SVG를 index.html에 인라인 삽입 (외부 파일·이미지 없음)
- 귀여움 문법: 큰 머리, 아래쪽에 몰린 작은 이목구비, 점 눈, ω 입, 발그레한 볼,
  오른쪽 귀 살짝 접힘 + 귀 밑 핑크 리본
- 표정 3종을 SVG 그룹으로 겹쳐 두고 CSS 클래스로 전환:
  - 기본(문제 중), 정답(눈웃음), 오답/시간초과(X자 눈 + 어질)
- 상단에서 말풍선으로 문제 프롬프트 출력 (기존 `#prompt`가 말풍선이 됨)

## 아케이드 연출 (전부 CSS 애니메이션, JS는 클래스 토글만)

| 상황 | 연출 |
|---|---|
| 정답 | 선택 버튼 팝 + `+점수` 플로팅 + 토끼 점프(눈웃음) |
| 오답 | 화면 살짝 흔들림 + 정답 버튼 강조 + 토끼 어질(X눈) |
| 콤보 배율 상승 | 🔥 배지 펄스 + 점수 칩 바운스 |
| 남은 시간 30% 이하 | 타이머 바 빨강 점멸 + 토끼 초조 모션 |
| 게임오버 | 토끼 위로 화면. 신기록이면 축하 점프 + ⭐ 반짝 |
| 접근성 | `prefers-reduced-motion`이면 흔들림·점멸 등 큰 모션 생략 |

- 애니메이션 트리거는 `state.phase`/`state.lastResult`/콤보 배율 변화를 game.js가
  감지해 클래스를 붙였다 떼는 방식. 규칙 판단 로직은 game-core.js에 이미 있으므로
  새 규칙 코드는 생기지 않는다.
- "시간 30% 이하" 판정은 `state.timeLeftMs / TIME_LIMIT_MS` 비율 계산 —
  표시 계층의 문턱값이므로 game.js에 둔다.

## 레이아웃 (모바일 세로 우선)

- 세로 순서: HUD(하트·점수 칩) → 타이머 바 → 토끼 + 말풍선 → (유동 여백) →
  보기 버튼(하단 엄지 존) → 피드백/다음 버튼
- 이지선다(compare): 좌우 큰 버튼 + 가운데 VS 알약
- 사지선다(arithmetic/equivalent): 같은 자리 2×2 그리드 (기존 `.quad` 방식 유지)
- 터치 타깃 최소 48px, PC/태블릿 가로에서는 최대 폭 ~28rem 중앙 정렬
